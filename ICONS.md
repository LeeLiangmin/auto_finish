# 图标文件说明

插件需要图标文件，请将图标放在 `icons` 目录下。

## 图标文件位置

图标文件应放在项目根目录的 `icons/` 文件夹中。

## 两种使用方式

### 方式1：使用统一图标（推荐）

将单个图标文件命名为 `icon.jpeg` 放在 `icons/` 目录：
```
icons/
  └── icon.jpeg
```

构建时会自动缩放为三个尺寸（16x16, 48x48, 128x128）。

### 方式2：使用分尺寸图标

分别提供三个尺寸的图标文件：
```
icons/
  ├── icon16.jpeg  (16x16 像素)
  ├── icon48.jpeg  (48x48 像素)
  └── icon128.jpeg (128x128 像素)
```

## 图标要求

- **输入格式**：JPEG 或 PNG（放在 `icons` 目录）
- **输出格式**：PNG（构建后自动转换到 `dist` 目录）
- **尺寸**：
  - 如果使用统一图标，建议使用较大的尺寸（至少 128x128）
  - 如果使用分尺寸图标，建议精确匹配对应尺寸
- **背景**：建议使用透明背景或纯色背景

## 自动转换

构建脚本会自动：
1. 从 `icons/` 目录读取图标文件
2. 转换为 PNG 格式
3. 调整到正确尺寸
4. 输出到 `dist/` 目录

## 依赖

图标转换需要 `sharp` 库，运行 `bun install` 会自动安装。

如果没有安装 `sharp`，构建脚本会尝试直接复制文件，但建议安装以获得更好的图片处理能力。
